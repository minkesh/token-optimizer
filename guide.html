Quick Install Start Here

One command installs everything: scripts, hooks, aliases, templates, and Graphify.

Run this in your terminal:

# Clone the toolkit
git clone https://github.com/multiply-marketing/token-optimizer ~/.token-optimizer-src
cd ~/.token-optimizer-src

# Run installer
bash install.sh

# Reload your shell
source ~/.zshrc  # or ~/.bashrc

Or manually copy the toolkit into any project:

# Copy optimized CLAUDE.md to your project
cp ~/.token-optimizer/templates/CLAUDE.md ./CLAUDE.md

# Install session hooks
mkdir -p .claude/hooks
cp ~/.token-optimizer/hooks/session-start.sh .claude/hooks/

# Install git hooks (auto-rebuild graph on commit)
cp ~/.token-optimizer/hooks/post-commit .git/hooks/post-commit
chmod +x .git/hooks/post-commit
After install, you get 4 new commands: newfeature · tokencheck · tokenstats · graphrebuild

Graphify Setup 71x Savings

The single highest-impact optimization. Instead of Claude grepping every file, it reads a compact knowledge graph.

01
Install Graphify
71x token reduction
▾

Graphify builds a persistent knowledge graph from your codebase. Code is parsed locally via AST — no source code leaves your machine.

# Install (note the double-y)
pip install graphifyy

# Verify
graphify --version
The PyPI package is graphifyy (double-y). graphify (single-y) is a different, unrelated library.
02
Build your first graph
one-time setup
▾

Run this in your project root. The first run costs tokens (semantic extraction), but every query after uses the compact graph instead of raw files.

# Build the graph from project root
cd /your/project
graphify .

# Outputs:
#   graphify-out/GRAPH_REPORT.md  ← Claude reads this
#   graphify-out/graph.json       ← persistent state
#   graphify-out/graph.html       ← interactive visualizer
After this, type /graphify . inside Claude Code to activate it for the session.
03
Install the Claude Code hook
auto-activates per session
▾

This PreToolUse hook fires before every Grep/Glob call. It tells Claude to check the graph first instead of scanning raw files.

# Add to .claude/settings.json
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Glob|Grep|Read",
        "hooks": [{
          "type": "command",
          "command": "echo 'graphify: GRAPH_REPORT.md exists. Consult it before raw file search.'"
        }]
      }
    ]
  }
}
04
Auto-rebuild on commit
git hook
▾

The post-commit hook rebuilds the graph automatically after code changes. AST-only rebuilds (code files) take under 5 seconds with no LLM tokens spent.

# Install the git hook
cp ~/.token-optimizer/hooks/post-commit .git/hooks/post-commit
chmod +x .git/hooks/post-commit

# Or use graphify's built-in hook installer
graphify hook install
Keep the graph fresh. A stale graph (3+ days old) gives Claude outdated structural info. The tokencheck command will warn you.

CLAUDE.md Architecture 62% Saved

CLAUDE.md is loaded on every single message. A 2,000-token CLAUDE.md costs 2,000 tokens × every message in every session. Keep it surgical.

01
The 3-tier architecture
target: <600 tokens

Structure your context in three tiers. Only Tier 1 loads every message. Tiers 2 and 3 load on demand.

TIER 1 CLAUDE.md loads every message
Project rules · model guidance · file map references
< 600 tokens — keep it lean
▼
TIER 2 docs/*.md loads on demand per task
docs/api-routes.mdonly loaded for API work
docs/schema.mdonly loaded for DB work
docs/testing-guide.mdonly loaded for test work
▼
TIER 3 graphify-out/GRAPH_REPORT.md queried per question
Knowledge graph — ~1,700 tokens vs ~100,000+ naive file scanning
Ready-to-use template included: ~/.token-optimizer/templates/CLAUDE.md
02
Monorepo: split CLAUDE.md per subdirectory
lazy loading

Subdirectory CLAUDE.md files only load when Claude navigates into that directory. Use this to scope context to where it's needed.

│ ├──frontend/CLAUDE.mdReact rules — only loads for frontend work

├──packages/
│ └──shared/CLAUDE.mdShared lib rules
└──docs/
  ├──schema.md

Session Habits Daily Practice

These habits compound. Each one is small. Together they cut usage by 60-90%.

🔄
Clear between unrelated tasks
Run /clear when switching to a completely different task. Stale context from the previous task gets re-read on every message.
🗜️
Compact before big operations
Run /compact when you're about to start something large. It squeezes the conversation to just decisions, changes, and open questions.
📦
Batch your questions
Instead of 3 follow-up messages, write one message with all 3 questions. Every separate message forces Claude to reprocess the entire history.
✂️
Request diffs, not full files
Add to your prompt: "Output as a diff, not the full file." A 300-line file rewrite costs 5–10x more than a targeted diff.
🎯
Reference by location, don't paste
Instead of pasting code, say: "In auth.ts, lines 42–58 — fix the race condition." Claude already has the file. Re-pasting is pure waste.
⚡
CLI over MCP tools
Use gh CLI instead of GitHub MCP. CLI returns targeted output. MCP dumps full JSON blobs into context — often 10–100x bigger.
🚀
Start every session with newfeature
The newfeature command generates an optimized session prompt, picks the right model, checks graph freshness, and copies it to your clipboard.
🧹
Don't thank Claude
Sending "Thanks!" or "That's perfect!" after a long session re-sends thousands of tokens of history for a response that adds zero value.

Model Selection Guide Up to 70% Cost Cut

Opus costs 5× more than Sonnet. Most daily tasks don't need it. The rule: start on Sonnet, escalate only when you need deep reasoning.

Model Use For Cost Level Switch Command
Haiku Renaming, formatting, quick lookups, docstrings, typos Low /model haiku
Sonnet Tests, standard edits, explaining code, daily work (80% of tasks) Medium /model sonnet
Opus Complex architecture decisions, tricky bugs, multi-file refactors High /model opus
opusplan Complex planning — uses Opus for plan, Sonnet for execution Medium-High /model opusplan
The newfeature script auto-recommends the right model based on what you're working on (detected from task description keywords).
Extended thinking is enabled by default and can burn tens of thousands of tokens per request on complex tasks. For simpler tasks, disable it: /config thinking false or lower the budget: MAX_THINKING_TOKENS=4000
